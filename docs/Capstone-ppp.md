# ASE-Capstone PPP

## Problem Domain

- Many people aren't familiar with terminals
  - difficult to jump in
- A game can teach basic concepts
  - give instant feedback

---

## Features and Requirements

1. A player character
   1. Can take actions such as move, open, attack, talk
   2. has health 
2. A terminal that acts as a controller
   1. user can type in actions that the character can take
   2. user can create scripts that can allow them to automate/optimize actions
3. Inventory system
   1. simulates a files and folders system
   2. User can gain items  
   3. User can use items

   
---
## Features and Requirements (Cont.)

4. Enemies
   1. AI determined by scripts similar to what the player may create for themself
   2. varying levels of difficulty/complexity of enemies
5. Combat system    
   1. User can damage and be damaged by enemies
   2. User can test knowledge of commands or efficiency of scripts against enemies
6. Main level
   1. combat encounters
   2. environmental hazards
   3. win state

---

## Sprints Timeline

1. Start on character and inventory system
2. Begin terminal system
3. Work on enemies and combat system
4. ensure terminal system is done and put together the features into a main level
